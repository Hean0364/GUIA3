Preguntas y Respuestas

1. ¿Qué se puede concluir a partir del ejemplo 1 y los ejercicios desarrollados hasta el momento?

El ejemplo 1 y los ejercicios muestran cómo se puede estructurar un documento XML bien formado. Es importante seguir las reglas de XML, como cerrar correctamente las etiquetas, no traslapar etiquetas, respetar mayúsculas y minúsculas, y utilizar atributos entre comillas. Además, se enfatiza que no es necesario apegarse a una DTD, ya que XML permite la creación flexible de etiquetas y atributos propios.

2. ¿Qué se puede concluir a partir del ejemplo 2 y los ejercicios desarrollados hasta el momento?

El ejemplo 2 y los ejercicios refuerzan la idea de que los atributos y elementos en XML deben usarse de manera lógica y coherente. Por ejemplo, una moneda o código telefónico son atributos del país, no de los departamentos. Esta diferenciación entre atributos y elementos también es clave para estructurar bien los datos en XML.

3. ¿Qué se puede concluir de los ejercicios de conversión de XML a JSON?

La conversión de XML a JSON facilita el uso de datos en diferentes contextos, ya que JSON es un formato ampliamente utilizado en aplicaciones web. Este ejercicio permite ver cómo se puede transformar un documento XML en un objeto JSON que puede ser manipulado mediante JavaScript.

4. ¿Qué se puede concluir de los ejemplos y ejercicios CSS?

El uso de hojas de estilo CSS aplicadas a XML permite personalizar la visualización de los datos estructurados en un navegador. Se destaca cómo el atributo display puede modificar cómo se muestran los elementos y cómo, a través de selectores CSS3, es posible mostrar atributos de las etiquetas XML en la pantalla, aunque con ciertas limitaciones.
